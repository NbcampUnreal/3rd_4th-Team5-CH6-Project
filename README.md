# TINYSCAPE (타이니스케이프)
> 작은 인간, 거대한 폐가, 살아남기 위한 협동 생존 액션

---

## 1. 프로젝트 개요 (Project Overview)
- **게임 제목:** TINYSCAPE (타이니스케이프)
- **장르:** 4인 멀티플레이 협동 생존 액션
- **플랫폼:** PC (Steam)
- **엔진/버전:** Unreal Engine 5.6
- **네트워크:** Steam 리슨 서버 기반

---

## 2. 게임 소개 (Game Description)
- **스토리:**  
플레이어는 저주에 침식된 폐가에서 생존하며, 거인과 몬스터의 위협을 피하면서 보금자리를 확보해야 합니다.

- **핵심 컨셉:**  
작아진 인간의 시점에서 바라본 폐가의 압도적인 스케일 공포와 협동 생존

- **게임플레이 핵심:**  
  - **생존:** 식량, 물 확보, 빛과 정신력 관리  
  - **전투:** 몬스터와 거인 상대, 협동 전투  
  - **탐험:** 폐가 곳곳 탐험, 재료 수집, 아이템 제작, 건축

## 4. 설치 및 실행 (Installation / How to Run)

### 양한아

<details>
<summary><b>1. 정적 데이터 관리 시스템</b></summary>

게임 내 모든 정적 데이터(아이템, 크래프팅 레시피, 건축물 레시피)를 통합 관리하는 5단계 아키텍처 구축.

**핵심 구조:**
- **GameplayTag → 데이터 구조체 → 데이터 에셋 → 시스템 설정 → 서브시스템**
- TMap 기반 O(1) 조회 성능
- 런타임 메모리 캐싱으로 디스크 I/O 제거
- Project Settings 통합 관리

**주요 성과:**
- 신규 데이터 추가 시간 80% 단축
- 코드 수정 없이 에디터 데이터 테이블에서 직접 편집 가능
- Listen Server 환경에서 복제 없이 로컬 캐시 기반 안정적 동작

**구조도 및 흐름도:**

![흐름도](./images/양한아/정적데이터관리시스템_데이터흐름.png)

</details>

<details>
<summary><b>2. 아이템 사용 및 효과 적용 시스템 (GAS 기반)</b></summary>

Gameplay Ability System을 활용한 데이터 주도 설계로, 최소 코드로 다양한 아이템 효과를 구현.

**카테고리별 최적화:**
- **소모품:** C++ 베이스 Ability + GEC 자동 스탯 분기
  - EffectTag → Attribute 자동 매핑
  - Multicast RPC 기반 몽타주 동기화 (네트워크 지연 보정)
  - 이동/피격 시 소비 중단 시스템
- **무기/도구:** GameplayEffect만으로 스탯 적용 (Ability 불필요)
  - SetByCaller 방식 자동 적용
  - 채취 태그 시스템 및 이벤트 기반 내구도 관리
- **방어구:** 공통 스탯 + 특수 효과 분리 (2개 GE)
  - 선택적 효과 적용 (피해 반사/감소, 이동속도)
  - Health 클램핑 처리

**주요 성과:**
- 아이템 추가 시간: 30분 → 1분 (30배 단축)
- 블루프린트 설정 최소화 (ItemID + GE 매핑만)
- 서버 권한 기반 치트 방지 + GAS Replication 자동 동기화

**아이템 시스템 이미지:**

![아이템 카테고리별 처리 시스템](./images/양한아/아이템사용시스템_카테고리별.png)

<details>
<summary><b>3. 기믹 영역 시스템 (VFX 포함)</b></summary>

GAS 기반 환경 효과 시스템으로, 독/전기 감전/속도 증감 등 다양한 기믹을 통합 관리.

**시스템 구조:**
- **GimmickZoneBase:** 다중 GameplayEffect 지원, 면역 시스템, 주기적 체크
- **TimeTickManager:** 서버 전용 타이머 서브시스템 (델리게이트 구독 패턴)

**기믹별 구현 (VFX 직접 제작 및 적용):**
- **독 영역:**
  - 직접 제작한 **Niagara 거품 파티클** 적용
  - **PostProcess 효과 C++로 직접 구현** (비네팅, 색조 보정, Bloom, DOF)
  - Fade In/Out 머티리얼 전환
- **전기 감전 영역:**
  - 직접 제작한 **Niagara 스파크 파티클** 적용
  - **PostProcess 깜빡임 효과 C++로 직접 구현** (Sin 파형 기반 FlickerValue 계산)
  - 해골 머티리얼 블링크 + 카메라 셰이크
  - 감전 사운드 효과 동기화
- **속도 증감 영역:**
  - Frankenstein/Mummy 머티리얼 독립적 Fade 전환
  - 액터별 TMap 관리로 동시 다중 액터 처리

**최적화 및 안정성:**
- Raw Pointer + IsValid() 조합으로 크래시 방지
- GameplayCue 재사용 시 초기화 로직으로 버그 예방
- 로컬 플레이어 전용 PostProcess 효과 분리

**주요 성과:**
- **Niagara VFX 직접 제작 및 통합** (기믹 시스템 전용)
- **PostProcess 효과 C++ 직접 구현** (독/감전 상태 시각화)
- 서버 권위 기반 + 클라이언트 로컬 효과 분리로 네트워크 안정성 확보
- 모듈화 구조로 신규 기믹 추가 용이

**※ 기믹 시스템의 Niagara VFX 제작 및 PostProcess C++ 구현은 모두 본인(양한아)이 직접 작업한 내용입니다.**
